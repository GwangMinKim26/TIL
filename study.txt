값(value)
- 표현식이 평가된 결과
- 더 이상 계산되거나 평가될 수 없는 프로그램의 가장 기본적인 데이터 조각

모든 값은 그 자체로 가장 단순한 형태의 표현식이지만, 모든 표현식이 값인 것은 아니다.
ex) 3+5(표현식), 8(표현식이자 값)

변수(variable)
- 값을 나중에 다시 사용하기 위해, 그 값에 붙여주는 고유한 이름
- 객체를 가리키는 이름

변수 할당(variable assignment)
- 표현식이 만들어낸 값에 이름을 붙이는 과정(연결?)

할당문 (Assignment Statement)
ex) degrees(변수 이름) =(할당 연산자) 36.5(표현식) -> 36.5라는 표현식(값)을 degrees라는 변수에 저장(할당)

변수명 규칙
- 영문알파벳, 언더스코어(_), 숫자로 구성
- 숫자로 시작할 수 없음
- 대소문자를 구분
- 내부 예약어는 변수로 사용할 수 없음

변수, 값 그리고 메모리
-메모리의 모든 위치에는 그 위치를 고유하게 식별하는 메모리 주소가 존재
-고유한 ID(메모리 주소) + 타입(type)+ 값(value)를 묶은 것을 객체라고 불린다.
-변수는 특정 개체를 가리키는 이름표
*변수는 메모리 주소를 가지지 않는다.

할당문 동작 순서
오른쪽 표현식 평가
왼쪽 변수명 확인
변수명과 결과값 연결(참조)


재할당 (reassignment)
-이미 값이 할당된 변수에 새로운 값을 다시 할당하는 것
-재할당은 이 변수가 가리키는 대상을 새로운 값으로 변경하는 행위

타입(type)
-변수나 값이 가질 수 있는 데이터의 종류를 의미
-2가지 요소로 구분 '값'과 '값에 적요할 수 있는 연산'
-데이터가 필요한 이유 -> 각 타입에 따라 가능한 기능과 연산이 다르기 때문

숫자형 데이터(numeric types)
int(정수형) : 소수점이 없는 숫자를 표현(양수, 0, 음수 포함)
float(실수 자료형) : 소수점이 있는 더 정밀한 숫자를 표현

지수 표현법
1.23  * 10^9 => 1.23e9, 3.14  * 10^-3 => 3.14e-3

시퀀스 타입(sequence types)
-여러개의 값들을 순서대로 나열하여 저장하는 자료형
-str, list, tuple, range
-인덱스(index):시퀀스 자료형에서 각 값의 위치를 식별하기 위해 부여된 고유한 번호
시퀀스 타입의 5가지 특징
순서(order): 값들이 순서대로 저장(정렬 X)
인덱싱(indexing): 각 값에 고유번호(인덱스)를 가지고 있으며 인덱스를 사용하여 특정 위치의 값을 선택하거나 수정할 수 있음
슬라이싱(Slicing): 인덱스 범윌ㅡㄹ 조절해 전체 데이터 중 원하는 부분만 값을 잘라내서 사용할 수 있음
길이(Length): len()함수를 사용하여 저장된 값의 개수(길이)를 구할 수 있음
반복(Iteration) : 반복문을 사용하여 각 값을 하나씩 순서대로 꺼내 사용할 수 있음

문자열(str)
-문자들의 순서가 있는 변경 불가능한 시퀀스 자료형
-이스케이프 시퀀스(Escape Sequence) : 역슬래시(\)와 문자를 조합해 특별한 기능을 수행

f-string
*f-string advanced
-문자열 내에 변수나 표현식의 결과를 손쉽게 삽입하는 강력한 방법

ex) print(f"안녕하세요, 제 이름은 {name}이고 나이는 {age}살 입니다.")

인덱스란?
시퀀스 자료형에서 각 갑의 위치를 식별하기 위해 부여된 고유한 번호
*인덱스는 0부터 시작함 (거리의 개념)

슬라이싱 사용 방법
-대괄호 []안에 시작 위치,끝 위치, 간격을 콜론으로 구분하여 지정
ex) my_sequence[start : stop : step]
start : 슬라이싱을 시작할 인덱스 (포함됨)
stop: 슬라이싱을 끝낼 인덱스 (포함되지 않음)
step : 몇 개씩 건너뛰며 값을 가져올지에 대한 간격

Remote Repository(원격 저장소)
코드 버전 관리 이력을 올라인 상의 특정 위치에 저장하여 여러 개발자가 협업하고 코드르ㄹ 공유할 수 있는 저장공간
local repository에 remote 명령어를 통해 로컬 저장소에 원격 저장소를 추가한다.
ex) git remote add origin remote_repo_url => origin(별칭, 별칭을 사용해 로컬 저장소 한개에 여러 원격 저장소를 추가할 수 있음)
git push origin master : 원격 저장소에 commit 목록을 업로드
원격 저장소에는 commit이 올라가는 것

Pull & Clone
git pull origin master : 원격 저장소의 변경사항만을 받아옴 (업데이트)
git clone remote_repo_url : 원격 저장소 전체를 복제(다운로드)

이미 Git에 올라가있는 파일은 gitignore를 할 수 없다

Revert & Reset
Git revert : 특정 commit을 없던 일로 만드는 작업-> 아예 없었던 일로 처리후 그 결과를 새로운 commit으로 추가함
git revert  <commit id> -> git log를 입력하고 commit 뒤에 있는 id를 입력
변경사항을 안전하게 실행 취소할 수 있도록 도와주는 순방향 실행 취소 작업
Git Reset : 특정 commit으로 되돌아 가는 작업
git reset [옵션] <commit id>  -> commit id까지 돌아가기
*되돌아간 commit 이후의 commit은 모두 삭제
--soft : 삭제된 commit 들의 기록을 staging area에 남김
--mixed : 삭제된 commit들의 기록을 working directory에 남김
--hard : 삭제된 commit들의 기록을 남기지 않음
**git remote -v 현재 내가 설정한 remote를 볼 수 있음
